## 🚀 Quick Start | البدء السريع

### 1️⃣ Basic Usage

```bash
# Fix lib/ directory
./scripts/fix_imports.sh

# Preview changes (dry run)
./scripts/fix_imports.sh --dry-run

# Fix with detailed output
./scripts/fix_imports.sh --verbose

# Generate report
./scripts/fix_imports.sh --report
```

### 2️⃣ Advanced Usage

```bash
# Fix all directories (lib + test + integration_test)
./scripts/fix_imports.sh --all

# Fix specific directory
./scripts/fix_imports.sh --path lib/models

# Fix test files only
./scripts/fix_imports.sh --test

# Combine options
./scripts/fix_imports.sh --all --verbose --report
```

### 3️⃣ Analysis

```bash
# Analyze imports
python3 scripts/analyze_imports.py

# Analyze specific directory
python3 scripts/analyze_imports.py lib/services
```

## 📐 Import Order Rules | قواعد الترتيب

### Standard Order

```dart
// 1️⃣ Dart core libraries
import 'dart:async';
import 'dart:convert';
import 'dart:io';

// 2️⃣ Flutter framework
import 'package:flutter/material.dart';
import 'package:flutter/services.dart';

// 3️⃣ External packages
import 'package:provider/provider.dart';
import 'package:isar/isar.dart';

// 4️⃣ Project imports
import 'package:apex_note/models/note.dart';
import 'package:apex_note/services/database_service.dart';

// 5️⃣ Relative imports
import '../controllers/notes_provider.dart';
import 'note_editor.dart';

// 6️⃣ Part directives
part 'note.g.dart';

// 7️⃣ Exports
export 'models/note.dart';
export 'models/note_type.dart';
```

## 🛠️ Tools Overview | نظرة على الأدوات

### 1. fix_imports.dart
**Main import fixer script**

**Features:**
- Sorts imports by category
- Removes duplicates
- Preserves comments
- Handles part/export directives
- Dry run mode
- Detailed statistics

**Usage:**
```bash
dart scripts/fix_imports.dart [OPTIONS]
```

**Options:**
- `--path PATH` - Target directory (default: lib)
- `--dry-run` - Preview without changes
- `--verbose` - Detailed output
- `--report` - Generate JSON report

### 2. fix_imports.sh
**Shell wrapper for easy execution**

**Features:**
- Multi-directory support
- Batch processing
- Help documentation
- Error handling

**Usage:**
```bash
./scripts/fix_imports.sh [OPTIONS]
```

**Options:**
- `--all` - Fix lib + test + integration_test
- `--test` - Fix test files only
- `--dry-run` - Preview mode
- `--verbose` - Detailed output
- `--report` - Generate report
- `--help` - Show help

### 3. analyze_imports.py
**Python analyzer for detailed analysis**

**Features:**
- Import statistics
- Duplicate detection
- Package usage analysis
- Issue reporting
- JSON export

**Usage:**
```bash
python3 scripts/analyze_imports.py [DIRECTORY]
```

### 4. import_fixer.config
**Configuration file**

Defines rules and options for import fixing.

## ⚙️ Configuration | الإعدادات

### analysis_options.yaml

```yaml
include: package:flutter_lints/flutter.yaml

analyzer:
  errors:
    directives_ordering: error

linter:
  rules:
    directives_ordering: true
```

```json
{
  "dart.lineLength": 80,
  "editor.formatOnSave": true,
  "editor.codeActionsOnSave": {
    "source.organizeImports": true
```

## 🎓 Best Practices | أفضل الممارسات

<div dir="rtl">

### ✅ افعل

1. **استخدم الترتيب القياسي** - اتبع قواعد Dart
2. **تجنب التكرار** - استيراد واحد لكل حزمة
3. **استخدم relative imports** - للملفات المحلية
4. **نظّم في مجموعات** - فصل واضح بين الأنواع
5. **راجع بانتظام** - شغّل الأداة دوريًا

### ❌ لا تفعل

1. **لا تخلط الأنواع** - لا تخلط dart: مع package:
2. **لا تكرر** - تجنب الاستيرادات المكررة
3. **لا تستخدم absolute paths** - للملفات المحلية
4. **لا تهمل التنظيم** - الفوضى تؤدي لأخطاء
5. **لا تتجاهل التحذيرات** - راجع flutter analyze

</div>

## 🐛 Troubleshooting | حل المشاكل

### Common Issues

#### 1. Permission Denied
```bash
chmod +x scripts/fix_imports.sh
```

#### 2. Dart Not Found
```bash
# Add Flutter to PATH
export PATH="$PATH:/path/to/flutter/bin"
```

#### 3. Python Not Found
```bash
# Install Python 3
sudo apt install python3  # Linux
brew install python3      # macOS
```

#### 4. Script Errors
```bash
# Check Dart version
dart --version

# Verify file encoding
file scripts/fix_imports.dart
```
