# Vault Reset & Destroy Feature

## Overview

تم إضافة ميزات جديدة لإدارة الخزنة:
1. **إعادة تعيين تشفير الخزنة** — إعادة تشفير كل الملاحظات بمفتاح جديد
2. **تدمير الخزنة** — حذف الخزنة نهائياً مع خيارين للتعامل مع المحتوى
3. **حماية من فقدان الوصول** — كشف إلغاء حماية الجهاز وتعطيل القفل تلقائياً

---

## 1. إعادة تعيين التشفير (Vault Reset)

### الفلو

1. إعدادات الخزنة → "إعادة تعيين التشفير"
2. شاشة تحذير + تحقق بالبصمة
3. إنشاء كلمة مرور جديدة (نفس `VaultPasswordPage` الأصلي — 8 أحرف، رقم، رمز، تطابق)
4. شاشة تقدم حقيقي مع progress bar (لا يمكن الخروج — `PopScope`)
5. عرض كود الاسترداد الجديد + تأكيد الحفظ

### العملية الداخلية

```
1. نسخ ملف القاعدة (sinan_notes.isar) كـ backup كامل
2. قراءة المفتاح الحالي (الخزنة مفتوحة بالبصمة)
3. جلب كل الملاحظات المشفرة من القاعدة
4. فك تشفير كل ملاحظة بالمفتاح القديم (مع تقدم مرئي)
5. مسح المفتاح القديم من الذاكرة فوراً
6. VaultService.setupVault(newPassword) → مفتاح جديد + كود استرداد جديد
7. إعادة تشفير كل ملاحظة بالمفتاح الجديد (مع تقدم مرئي)
8. كتابة كل ملاحظة للقاعدة
9. مسح المفتاح الجديد من الذاكرة
```

### الحماية من الفشل (Atomic Safety)

- **قبل أي تعديل**: نسخ ملف القاعدة بالكامل كـ backup
- **لو فشلت العملية** (crash/خطأ): استعادة القاعدة القديمة فوراً عبر `_restoreFromBackup`
- **لو نجحت**: الاحتفاظ بالنسخة القديمة 15 يوم كـ fallback
- **تنظيف تلقائي**: `cleanExpiredBackups()` عند كل فتح للتطبيق يحذف النسخ الأقدم من 15 يوم

### أداء الـ UI (Non-blocking)

- `_yieldToUI()` — `Future.delayed(Duration.zero)` بعد كل emit لإعطاء الـ event loop فرصة لرسم الـ frame
- yield كل 2-3 ملاحظات لتقليل overhead مع ضمان تحديث مرئي
- `Future.delayed(16ms)` قبل `setupVault` لأنها عملية ثقيلة (PBKDF2 100k iterations)
- `Future.delayed(100ms)` قبل `executeReset` لضمان رسم شاشة التقدم أولاً

## 2. تدمير الخزنة (Vault Destroy)

### الخيارات

#### فك التشفير وتدمير (Decrypt & Destroy)
- فك تشفير كل الملاحظات المقفلة بالمفتاح الحالي
- نقلها للملاحظات العادية (`isLocked = false`)
- حذف الخزنة ومفاتيح التشفير (`VaultService.clearVault()`)
- تحديث الصفحة الرئيسية (`refreshAllNotes`)

#### تدمير مع المحتوى (Destroy with Content)
- حذف كل الملاحظات المقفلة نهائياً من القاعدة
- حذف الخزنة ومفاتيح التشفير
- تحديث الصفحة الرئيسية

### الحماية (3 طبقات)

1. **Checkbox تأكيد**: "أفهم أن هذا الإجراء لا يمكن التراجع عنه" — الزر معطّل بدونه
2. **تحقق بالبصمة**: بعد التأكيد، يُطلب التحقق بالبصمة (لو الجهاز يدعمها)
3. **VaultResetGuard**: يمنع `LockedNotesScreen` من الخروج أثناء البصمة

---

## 3. حماية من فقدان الوصول (Biometric Removed Detection)

### المشكلة
لو المستخدم ألغى كل وسائل المصادقة من الجهاز (بصمة + PIN + كلمة مرور)، التطبيق يصبح غير قابل للفتح.

### الحل

#### قفل التطبيق (Splash Screen)
- عند فتح التطبيق: لو `hasBiometrics() == false` و `isAppLockEnabled` → تعطيل القفل تلقائياً
- عرض رسالة: "تم تعطيل قفل التطبيق — لا توجد حماية على الجهاز"

#### الخزنة (Vault Entry)
- لو البصمة مفعّلة لكن `hasBiometrics() == false` → تعطيل البصمة تلقائياً
- توجيه المستخدم لشاشة كلمة المرور (لا يزال يستطيع الدخول بكلمة المرور أو كود الاسترداد)

#### عمليات التدمير
- لو الجهاز بلا حماية → تخطي البصمة وتنفيذ مباشر (المستخدم داخل الخزنة أصلاً)

---

## 4. إصلاح إغلاق الخزنة عند البصمة

### المشكلة
عند طلب البصمة، نافذة النظام تظهر فوق التطبيق → التطبيق يدخل حالة `paused` → `LockedNotesScreen.didChangeAppLifecycleState` يخرج من الخزنة.

### الحل
- `VaultResetGuard.isActive` — static flag في `vault_reset_service.dart`
- `LockedNotesScreen` يتجاهل lifecycle events طالما الـ guard مفعّل
- يُفعّل قبل البصمة ويُوقف عند dispose أو فشل البصمة
